# 来源评分与引用校验设计

这篇讲 `deepresearch/agents.py` 里的两个 Agent：

- `SourceEvaluatorAgent`
- `CitationVerifierAgent`

## 为什么需要来源评分

DeepResearch 不是搜到什么都信。

不同来源可信度不一样：

- 官方文档通常更权威
- 论文适合支持架构和趋势分析
- GitHub 适合观察生态和实现
- 博客和新闻只能作为旁证

所以要评分。

## 当前评分维度

`SourceEvaluatorAgent` 当前按四个维度评分：

```text
authority   权威性
freshness   新鲜度
relevance   相关性
risk        风险
```

最后得到：

```text
final
```

这个分数会进入报告。

## 为什么需要引用校验

Agent 最容易出的问题是：

> 结论写得很像真的，但来源并不支持。

所以需要 `CitationVerifierAgent`。

当前版本会把每个来源变成一个 claim，并标记：

```text
supported
weak
```

现在还是来源级校验。

未来要升级成逐句校验：

```text
报告中的每个结论
-> 找支持来源
-> 判断 source snippet 是否真的支持
-> 不支持就标 weak
```

## 面试话术

可以这样讲：

> 我把来源评分和引用校验拆成两个 Agent。Evaluator 负责从 authority、freshness、relevance、risk 四个维度评估来源质量；Verifier 负责把来源和 claim 绑定，判断结论是否有证据支持。这样报告不是单纯由模型生成，而是尽量让每个关键结论都能追溯到来源。
