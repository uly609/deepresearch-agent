# 上下文管理设计

这篇讲 `deepresearch/context_manager.py`。

## 为什么 Agent 需要上下文管理

很多新手会以为：

> 把所有历史都塞进 prompt 就行。

但 Agent 跑长任务时，这样一定会乱。

因为上下文里有不同类型的信息：

- 当前任务目标
- 最近步骤
- 搜到的来源
- 已经验证过的 claim
- 可以丢弃的旧日志
- 不能丢的关键证据

这些不能混在一起。

## ContextManager 管什么

当前 `ContextManager` 管这些字段：

```text
task_goal          当前任务目标
working_notes      工作记忆
evidence_pool      证据池
verified_claims    已验证结论
discarded_notes    被丢弃的旧笔记
```

你可以把它理解成 Agent 的资料夹。

## evidence_pool 为什么重要

DeepResearch 最重要的是证据。

如果模型生成了一个结论，但找不到来源支持，就容易幻觉。

所以项目里把来源集中放进：

```python
evidence_pool
```

后面引用校验、报告生成、上下文压缩都可以围绕证据池做。

## compact() 是什么

`compact()` 会把上下文压成一段短文本。

现在还只是简单版本。

以后接 LLM 时，它可以用来控制 prompt：

```text
目标
最近笔记
关键证据
已验证 claim
```

这就是上下文压缩的雏形。

## 面试话术

可以这样讲：

> 我没有把所有历史直接拼进 prompt，而是设计了 `ContextManager`，把任务目标、工作记忆、证据池、已验证 claim 分开管理。这样后续接 LLM 时，可以只把当前决策需要的信息压缩进 prompt，同时保证关键证据不会被丢掉。
