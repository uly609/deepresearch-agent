# 怎么学这个项目

这份文档按“新手能讲出来”为目标，不要求你一上来就懂所有代码。

学习顺序建议是：

1. 先把项目跑起来
2. 顺着一条请求看主链路
3. 每次只看一个模块
4. 每个模块都回答三个问题
5. 最后压成自己的面试话术

## 第一步：先跑起来

进入项目：

```bash
cd "/Users/ntroi/Downloads/ai 社区"
```

运行普通 Agent：

```bash
make run
```

运行评估：

```bash
make eval
```

语法检查：

```bash
make check
```

你先不用追求完全理解，先知道它真的会输出：

- 执行事件
- task id
- report 路径
- Markdown 报告

## 第二步：按主链路读

不要一上来乱点文件。

按这个顺序看：

1. `run_agent.py`
2. `deepresearch/cli.py`
3. `deepresearch/runtime.py`
4. `deepresearch/langgraph_loop.py`
5. `deepresearch/tools.py`
6. `deepresearch/rag.py`
7. `deepresearch/vector_store.py`
8. `deepresearch/agents.py`
9. `deepresearch/models.py`
10. `deepresearch/task_state.py`
11. `deepresearch/run_store.py`
12. `deepresearch/context_manager.py`
13. `deepresearch/security.py`
14. `deepresearch/checkpoint.py`
15. `deepresearch/eval_harness.py`

为什么这么排？

因为这就是一条请求实际走过的路径。

## 第三步：每个模块只回答三件事

每看一个文件，都强制自己回答：

1. 这个模块解决什么问题？
2. 它现在是怎么做的？
3. 为什么要单独拆出来，而不是写在一起？

比如 `tools.py`：

- 解决什么问题：统一管理外部工具
- 怎么做：定义 `SourceConnector` 和 `ToolRegistry`
- 为什么拆出来：后面接 GitHub、arXiv、MCP 时不用改 workflow

## 第四步：先讲清楚，不要先追求高级

你现在最重要的目标不是“项目多厉害”，而是能讲清：

> 一条研究问题进来后，Agent 怎么拆解、怎么找来源、怎么评分、怎么校验、怎么生成报告、怎么保存结果。

这条讲顺了，后面再加真实 API、LLM、RAG、MCP 才有意义。

## 第五步：建议你亲手改的小点

等你看完基础链路以后，可以亲手改这些点：

1. 在 `OfficialDocsConnector` 或离线 Connector 里加一个新来源
2. 在 `ReflectionAgent` 里加一个新缺口检查，比如 `browser`
3. 在 `SourceEvaluatorAgent` 里调整评分权重
4. 在 `ReportAgent` 里改报告结构
5. 在 `EvalHarness` 里加一个测试题

这些改动都不大，但能帮你把项目变成自己的。
